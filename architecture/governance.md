# Governance

## Architectural Decisions

Significant architectural decisions are documented as ADRs.

## Repository

Architecture artifacts are maintained in Git.

## Documentation Style

Architecture documentation is lightweight, text-first, and version-controlled.

## Technology Direction

Preference is given to:
- serverless architectures
- event-driven communication
- managed cloud services
- operational simplicity

## Change Management

Target-state and roadmap documents are updated when major architectural changes occur.

## ADR Lifecycle

ADRs are immutable historical records.
New decisions supersede previous ADRs rather than modifying them.

## Repository Ownership

The repository is maintained by the system creator and operator.

## Documentation Scope

This repository documents architectural decisions and high-level system design.
Implementation details belong to source repositories.
